# OQ-08 检索升级路径分段：编辑器上线到 C，D 延后到 LSP 接入

`requirements.md` OQ-08 原计划"编辑器（F-11）上线后依次升级 C（向量 RAG）→ D（LSP/AST）→ E（混合检索）"。现修订为：编辑器上线只触发到 **C**，**D 延后**到 LSP 真正接入时（即编辑器从中等能力边界升级到完整能力边界时），E 跟随 D。

**背景**：grill 会话定 F-11 编辑器能力边界为"中等：多行+行号+undo/redo+查找替换+tree-sitter 语法高亮"——**不含 LSP**。D 阶段依赖 LSP，若保留原计划则编辑器上线即必须接 LSP，与中等边界冲突。

**理由**：把 D 与编辑器中等边界解耦，避免编辑器上线就有硬依赖 LSP。tree-sitter AST 在中等边界已可用，可被 C 阶段复用（结构化切片喂向量库），不必等 D。分段后每阶段触发条件清晰：C = 编辑器上线，D = LSP 接入，E = D 后。

**备选（被拒）**：保留原计划一次性 C→D→E — 与中等边界冲突，需编辑器上线就接 LSP，等于回到完整能力边界，与 grill 决策矛盾。

**后果**：`doc/design/architecture.md` 6.3 `ContextProvider` trait 描述需补"C 阶段实现复用 tree-sitter AST 切片喂向量库"。`requirements.md` OQ-08 状态需更新为分段路径。详见 `doc/design/editor-design.md` 第 7 节。
